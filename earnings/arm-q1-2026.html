⚠ 指引數字上超預期，股價為何仍下跌？

Goldman Sachs 直接指出："investor expectations were very elevated heading into the quarter, given recent bullish targets at ARM's recent investor event, as well as positive market sentiment on the growth prospects for CPUs." 機構投資人的「真實期望值」遠高於官方 Street 共識，「略超共識」等同於「低於買入倉位的預期」，觸發獲利了結。EPS 環比從 Q4 的 $0.60 降至 $0.40（−33%），且 AGI CPU 首批出貨收入確認要到 Q4 FY27 才入帳（約 $9,000 萬），讓近期催化劑顯得不足，難以支撐約 100x 遠期本益比的估值。

03 — 核心議題：財報重點與深度分析
需求爆發 AGI CPU：六週需求翻倍，但收入後移 FY28

自 3 月 24 日「Arm Everywhere」發表會後短短六週，AGI CPU 客戶需求已從 $10 億翻倍至超過 $20 億（橫跨 FY27+FY28），來源涵蓋既有客戶加碼及全新客戶。目前維持 $10 億出貨目標，積極擴大供應鏈（wafer、記憶體、封裝、測試設備）。Meta 為首席合作夥伴，SAP、Cloudflare、F5、SK Telecom、OpenAI、Cerebras、Rebellions 等均已宣佈部署計畫。在生態系支持層面，公司提前向 TSMC、三星、Synopsys、Cadence、AWS、Google、Microsoft、NVIDIA 等超過 50 家夥伴溝通矽晶片策略，全數表態支持——公司已售罄且有人詢問「多快能拿到貨」，本身就是最好的市場驗證。

收入兌現時間表

首筆實際出貨收入落在 Q4 FY27，金額約 $9,000 萬。全年 FY27 貢獻極低，$20 億需求大多落在 FY28。需求翻倍雖振奮人心，但近期財務催化劑不足是盤後下跌的核心原因之一。

為何部署快速？

客戶可直接向 Supermicro、Lenovo、ASRock 購買成架 rack，無需自行整合。加上多數客戶既有 ARM 軟體已完成——「軟體工作已經做完了，拿到 rack 就能部署」，大幅降低部署摩擦。

持續翻倍 資料中心 Royalty：連續翻倍，FY27 預期延續

資料中心 Royalty 連續 YoY 翻倍，管理層預計 FY27 仍將再次翻倍。驅動力橫跨三大超大規模雲服務商：AWS Graviton（與 Trainium、Nitro 並行）、Google Axion（取代 x86 主機處理器，效能提升 80%）、Microsoft Cobalt（Azure 高效能運算）。NVIDIA Vera（88 cores，Agentic AI 專用）亦全面採用 ARM，DPU/SmartNIC 中 ARM 市佔近 100%。頂尖超大規模雲服務商 ARM 架構佔比目前達約 50%。

Google Axion 案例：量級差距而非小幅改善

在 Google Cloud Next，Google 宣佈 TPU 8t（訓練）與 TPU 8i（推論）均改用自製 Arm Axion CPU 取代 x86 主機處理器，在相同功耗下效能提升 80%。這不是邊際改善，而是架構優勢的量級體現，也代表 ARM 在最核心的 AI 訓練基礎設施中已站穩位置。

架構競爭 CPU 核心數倍增、Agentic AI 工作負載，與 ARM vs x86 競爭格局

電話會議中，分析師詢問 CPU:GPU 比例問題。管理層的回答點出了一個更重要的框架：從晶片數量看比例意義有限，CPU 核心數才是關鍵指標。 NVIDIA Vera Rack（256 顆 Vera CPU × 88 cores = 22,528 核心，200 千瓦液冷機架）的公告就是這個邏輯的極致體現——管理層坦言，4 倍 CPU 需求成長的預估可能偏保守。

ARM 商業模式的核心邏輯：ASP 升級即收入升級

ARM 收取的是「每顆晶片的 Royalty」（約晶片售價 1~2%），而非「每個核心」。核心數增加的好處是間接但真實的：晶片 ASP 隨核心數等比上升，ARM 收到的絕對金額同等成長。 你可能不會看到多出三倍的晶片，但晶片會更貴——這正是為何 2030 年資料中心 CPU TAM 能從 $1,000 億膨脹到 $1,250 億，不靠出貨量，靠每顆晶片的價值上升。

為何高核心數對 ARM 架構特別有利

ARM 的 RISC 架構天生為高效率設計，核心面積小、功耗低。一顆 512 核心的 ARM CPU 在物理上完全可行；x86 要達到同等核心密度，功耗和面積代價大得多。GPU（Blackwell/Rubin）受 reticle 限制難以持續擴大，CPU 核心數卻可能從目前 136 倍增至 256、512 甚至更多。

Agentic AI 工作負載：完全契合多核心 ARM

CFO Jason Child 指出：每個 agent 的工作流程是發出請求、等待回應、排程下一步——高度非同步、低跨核通訊，各 agent 之間幾乎互相獨立。這和需要大量跨核協作的傳統 HPC 完全不同。因此同時跑 10,000 個 agent，你需要的是「10,000 個夠用的獨立核心」，而非「100 個超快的核心」。ARM 的優勢不在於單核速度比 x86 快，而在於同樣功耗預算下能塞進更多核心——agentic 工作負載恰好需要的正是更多核心。

對 ARM 財務的時間軸意涵

近期（FY27）：主要受益仍是 Neoverse IP royalty 繼續翻倍，AGI CPU 收入 Q4 才入帳，核心數升級的 ASP 效應尚未顯現於財報。

中期（FY28-29）：供應鏈解決後，AGI CPU 核心數升級帶動 ASP 上升將直接反映在 royalty 收入，GS 的正常化 EPS $2.50 估算在此情境下可能偏保守。

長期（FY30-31）：若 ARM 資料中心 CPU 市佔達多數，且核心數從 136→256→512 持續倍增，ASP 與 royalty 的複合成長效應將是實現 $250 億收入目標的關鍵結構性引擎。

生態系轉型 ARM 軟體生態系：瓶頸還是長期尾風？
生態系問題需拆分三層，性質完全不同

ARM CPU 的生態系問題並非鐵板一塊。Layer 1（OS / 工具鏈）幾乎已解決；Layer 2（Runtime / 中介軟體）約 90% 成熟；Layer 3（應用程式層）則存在明顯的新舊工作負載分歧，需分開評估。

x86 護城河的本質轉變

x86 的真正護城河從來不是效能，而是數十年累積的軟體生態系鎖定。但 SAP 宣佈把核心資料庫及業務應用遷移到 ARM（先 Graviton，再 AGI CPU）是一個關鍵轉折——SAP HANA 原本被認為是 ARM 最難攻克的堡壘。這個訊號的意義類似 Microsoft Office 遷移 ARM Mac 在個人電腦領域的地位轉折點：即便最 legacy 的企業應用也開始鬆動了。

✓ 幾乎已解決的部分

Layer 1：OS 與底層工具鏈

Linux kernel ARM64 支援自 2012 年起完整。GCC、Clang、LLVM 對 ARM 的最佳化非常成熟，對資料中心新建工作負載已不是瓶頸。

AI / ML 框架與雲原生服務

PyTorch、TensorFlow ARM 支援完整。Docker/Kubernetes multi-arch image 已是業界標準。CUDA 生態系不依賴 CPU 架構，頭節點換 ARM 對開發者無感。Rene Haas：「軟體的工作已經做完了，客戶拿到 rack 就能部署。」

企業應用遷移（SAP 為關鍵里程碑）

SAP 宣佈遷移到 ARM，Cloudflare 全球網路部署 ARM。3~5 年遷移週期，工具路徑成熟，速度是變數而非可能性。

✗ 仍存在摩擦的部分

Windows Server / on-premise 企業系統

主流 ISV 軟體認證仍以 x86 為主。大型企業核心系統遷移週期 3~5 年以上，組織風險容忍度建立需要時間——這是 x86 最後的護城河。

AVX-512 特定優化

針對 x86 向量指令集手工優化的數值計算庫需重新調整。ARM 的 SVE2 理論上更優雅，但生產環境工具成熟度仍在追趕中，對特定 HPC workload 仍是真實摩擦。

中國市場地緣政治變數

ARM 中國 JV 狀況複雜，地緣政治因素可能影響中國客戶長期採用意願——Morgan Stanley downside risk 特別點名的隱性風險。

最長期但最根本的因素：誰在寫新的程式碼？

AWS Graviton 實例比同規格 x86 便宜 20~40%，已成為雲端新建工作負載的預設選擇。今天被訓練出來的開發者，很多人的第一個生產環境就是 ARM。 十年後，「我的程式只在 x86 上測試過」這句話會越來越少出現。Apple Silicon 提供了最好的參考：2020 年切換時市場普遍擔憂生態瓶頸，兩年內絕大多數專業軟體完成原生化。這個複利效應是軟體生態從「潛在瓶頸」轉化為「長期結構性尾風」的根本機制。

工作負載類型生態系現狀遷移週期對 ARM 的意義

壓力持續 手機 & Edge AI：ASP 升級抵消量的下行

手機市場 Unit 出貨趨近負成長，低端市場壓力明顯，FY27 全年預計「持平或略為負成長」。但 Armv9 持續滲透高端機型 + CSS 帶動 ASP 提升，部分抵消量的影響，讓手機 royalty 仍維持正成長。汽車（ADAS/自動駕駛）持續雙位數成長，市佔持續擴大，為 Edge AI 增添穩定支撐。

04 — 盤後下跌 ~7% 的原因分析

⬇ 盤後一度下跌約 7%，從初步上漲急轉直下

01
Royalty 收入低於市場預期（−4.1% vs Street）——本季數字最差的分項
Royalty $671M 低於 Street $700M（−4.1%）及 GS $688M（−2.4%），是本季唯一明顯 miss 的分項。雖然資料中心 royalty 翻倍強勁，但手機低端市場疲軟拖累效應大於預估。Royalty 比 licensing 更具可預測性，其不如預期對長期成長能見度有負面暗示。
02
進場前市場預期已「過度樂觀」——GS 直接點名
Goldman Sachs 直接指出："investor expectations were very elevated heading into the quarter." 機構的「真實期望值」遠高於官方 Street 共識，「略超共識」等同於「低於買入倉位的預期」，觸發獲利了結。
03
AGI CPU 收入確認大幅後移至 FY28
需求雖翻倍至 $20 億+，但首筆實際出貨收入落在 Q4 FY27，金額僅約 $9,000 萬，全年 FY27 貢獻極低。$20 億需求大多落在 FY28，讓近期財務催化劑不足以支撐約 100x 遠期本益比的估值。
04
EPS 環比大跌 −33%（雖超預期，但對比 Q4 仍為心理衝擊）
Q1 EPS 指引中值 $0.40 雖大幅超出 Street 共識 $0.36（+12%），但相比 Q4 實際的 $0.60 仍環比下跌 −33%。絕對值的下滑讓習慣「線性成長敘事」的動能型投資人感到不安，特別是在股價已大漲 80~90% 年初至今的背景下。
05
供應鏈限制 + 手機市場確認疲軟
管理層坦承目前仍在爭取 wafer、記憶體、封裝及測試設備的增量供應，$10 億出貨目標面臨供應瓶頸。FY27 手機 Unit「預計持平或略為負成長」被明確點名，低端市場壓力持續。

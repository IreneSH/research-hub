03 重大商業進展與公司戰略敘事
🔶 Oracle Project Jupiter — 兩步公告、同一框架，完整時序解析

【第一步：4月14日 — 框架協議（母合約）→ 股價+22%，推升至4/24歷史新高$242】
Oracle與Bloom簽署主服務協議（Master Services Agreement），Oracle承諾採購最高2.8 GW的Bloom燃料電池系統，其中初始1.2 GW已完成簽約並開始部署，支援Oracle在美國各地的雲端項目。Oracle同時獲得認股權證，可以約$4億購買Bloom股票，利益深度綑綁。

【第二步：4月27日（財報前夜）— Project Jupiter子項目具名揭露】
Oracle、BorderPlex Digital Assets與Bloom聯合宣布，新墨西哥州Doña Ana County的AI園區「Project Jupiter」將完全採用Bloom燃料電池，裝機最高2.45 GW，取代原規劃的天然氣渦輪機與柴油發電機，整合為單一離網微電網。這是4月14日2.8 GW框架協議下首個具名的超大型子項目——把母合約的主要用途「具名化」，讓市場對訂單實現路徑更加清晰，並為隔日財報增添額外催化劑。

環保優勢：Bloom方案較原CCGT渦輪機可減少NOₓ排放約92%，且幾乎零耗水，有效化解社區阻力——這也是KR Sridhar在法說會上反覆強調「社區友善」（community-friendly）的核心背景。

KR Sridhar指出，Oracle案雖備受媒體關注，但Bloom目前數據中心backlog中逾半數來自其他超大規模雲端業者（hyperscalers）、新興雲端（neoclouds）及共置業者（co-location providers），顯示商業需求已高度多元化，並非單一客戶依賴。Bloom的戰略正從「展示期」進入「複製期」。

戰略要素 說明
製造產能策略 現有廠房最大產能達5 GW/年，採用「Copy Exact」模型持續擴產，以「每季數百MW」為單位連續增加，而非傳統一次性大型擴廠。
安裝時間突破 導入預製模組化skid設計，現場安裝時間縮短接近一個數量級（10倍），可快於任何競爭技術完成百MW級項目部署。
服務合約 100% attach rate（每筆產品銷售必附服務合約），數據中心客戶平均合約期限10–15年，提供高能見度的年金型收入。
自動化製造 產量從200 MW擴增約10倍至近2 GW，工廠人力保持不變——完全依靠自動化而非勞動力增員，解決供應鏈勞動成本風險。
新任CFO Simon Edwards，前任職Groq（AI推論晶片），系統工程背景，具備軟體公司CFO與製造擴張雙重經驗。
國際佈局 當前80–20法則，美國主導；歐洲因能源政策（俄烏/卡達天然氣）轉型受阻而有所延遲，但公司視為「時間問題而非可能性問題」。
04 法說會 Q&A 重點摘要
Q 隨著規模擴展到3/4/5 GW，毛利率的長期路徑如何思考？ JP Morgan — Mark Strouse
CFO Simon Edwards強調：當前季度的毛利大幅擴張主因是收入成長遠超過去幅度，公司透過規模效益、製造費用攤銷改善及持續供應鏈/工程降成本，推進毛利率。公司不提供長期毛利指引，但表示降成本為Bloom的核心DNA，未來每年持續雙位數降成本。
Q 服務合約期限是否有拉長？數據中心合約有無10年以上？ JP Morgan — Mark Strouse
KR Sridhar確認：(1) 所有交易均具備100% service attach rate，沒有例外；(2) 數據中心合約平均期限已達10–15年，高於過去C&I業務水準，將成為公司年金型收入的重要支柱。
Q 在其他替代方案成本上漲環境下，Bloom的定價策略？ Morgan Stanley — David Arcaro
KR Sridhar拒絕以競爭者定價作為benchmark，強調Bloom創造的是完全不同的「數位電力」價值——消除老舊備用基礎設施、提供更高可靠度與更快部署速度。公司不比較渦輪/引擎定價，而是以「創造客戶與Bloom雙方共贏」為定價邏輯。
Q 供應鏈如何支撐接近10倍的快速擴產？勞工瓶頸風險？ Morgan Stanley — David Arcaro
KR Sridhar指出最關鍵差異：工廠從200 MW/年擴至近2 GW/年，現場員工人數「完全相同」，靠的是全面自動化與員工技能升級，而非新增人力。供應鏈夥伴採用相同「Bloom Way」理念，提前佈局庫存與訂單。管理層對供應鏈風險表示高度信心。
Q 超過5 GW後的產能擴張是否需要新廠房？觸發條件為何？ RBC Capital Markets — Chris Dendrinos
KR Sridhar確認：超越5 GW必然需要新建廠房，Bloom已視之為「常規營運」而非一次性事件。新廠房興建時機取決於市場需求節奏，公司目標是「點亮整個星球」，5–6 GW/年遠不是終點。
Q Inference需求（非Training）對Bloom的意義？競爭者「橋接電力」方案的挑戰？ Evercore ISI — Nick Amicucci
▍什麼是「橋接電力」（Bridge Power）？
在AI數據中心電力需求爆發初期，超大規模雲端客戶面臨一個現實困境：永久性電網連接與大型發電廠建設通常需要數年時間，但GPU廠房必須盡快上線。「橋接電力」方案因此出現——即採用燃料電池、小型渦輪機或移動式發電設備作為過渡性臨時電源，待電網接入就緒後再切換至正式供電。這個概念意味著客戶把Bloom等公司的產品定位為「等待Superman（電網）來救援的臨時替代品」。

▍Bloom的立場演變：從「橋接」到「永久取代」
KR Sridhar表示，過去客戶確實以橋接電力的邏輯來接洽Bloom，他當時的回應是：「我們很樂意賣給你一座橋——因為Superman根本不會來。」如今這個對話已經不復存在——客戶現在直接要求完全離網（off-grid）的永久解決方案，不再考慮日後接回電網。

▍Inference vs. Training的電力意涵
KR Sridhar指出：Inference在總GW需求上將遠大於Training，但其部署位置分散在邊緣端（貼近人口密集區），與大型Training數據中心集中建在偏遠地區的模式截然不同。在市區或郊區部署Inference設施，社區對空氣污染、噪音的容忍度更低，使得Bloom的清潔、靜音、零排放特性競爭優勢更加突出。
Q 800V DC直接供電、無電池/無變壓器的全棧方案商業化進程？ UBS — Manav Gupta
▍什麼是800V DC架構？為何重要？
傳統數據中心採用415V交流電（AC）進場，經過多次降壓轉換（AC→低壓AC→54V DC→GPU電壓）才能抵達運算核心，每個轉換步驟都有能量損耗，且需要大型變壓器、整流器、大量銅線。NVIDIA在新一代Blackwell/Rubin Ultra平台（Kyber架構）中提出800V DC直流高壓架構（HVDC），把轉換步驟從四步壓縮至兩步（800V→6V→<1V），大幅提升傳輸效率並減少銅材用量。

▍Bloom的天然優勢
KR Sridhar確認：Bloom燃料電池本身輸出即為直流電，加上超級電容可直接輸出800V DC，無需額外整流器或大型變壓器——即「100% Bloom、一站式全棧電力方案」，完美對接NVIDIA新架構。KR表示此轉換是「必然趨勢」，因為全球銅材與大型變壓器供不應求，客戶一旦採用便不會回頭。

05 對 MTAR Technologies 的投資含義
MTAR Technologies (NSE: MTARTECH)
Bloom Energy唯一印度Hot Box Assembly供應商 · Clean Energy佔MTAR總營收55–65%

MTAR Technologies是Bloom Energy在印度的唯一critical供應商，專責生產固態氧化物燃料電池的核心組件——Hot Box Assembly（熱箱組件），在Bloom的採購需求中佔60–70%的份額（Wallet Share）。隨著Bloom本次大幅上調全年指引並宣布Oracle 2.45 GW超大訂單，MTAR的訂單能見度與成長力道均受到直接且正面的衝擊。

關鍵換算公式（Motilal Oswal估算）：

Bloom Energy每1 GW的訂單 → MTAR訂單流入約 ₹900–1,100億（約$108–132M）
Oracle Project Jupiter 2.45 GW → 潛在MTAR累積訂單流入約 ₹2,200–2,700億（按交付期分批認列）
Bloom FY26 Revenue中位數$3.6B對應產量（粗估）→ MTAR FY26–28 Revenue CAGR目標 40%
①

Bloom指引上修直接驅動MTAR訂單能見度大幅提升

Bloom全年Revenue指引從$3.1–3.3B上修至$3.4–3.8B（中位數+12.5%），且Q2 Revenue≥Q1的指引意味著全年出貨斜率比原先更陡。由於MTAR以Hot Box為核心出貨，Bloom的Revenue加速即等比轉換為MTAR的出貨量及新訂單流入。MTAR截至2025年12月底的訂單簿已達₹2,395億，管理層目標FY26底達₹2,800億（為FY25營收的3.5倍）。

②

Oracle Project Jupiter為MTAR帶來多年期超大型訂單觸媒

Oracle 2.45 GW的Project Jupiter是全球迄今最大的Bloom單一項目，且屬「100% Bloom」方案。MTAR作為Hot Box Assembly唯一供應商，理論上對應約₹2,200–2,700億的累積訂單流入，相當於MTAR估計FY26營收的1.6–2.5倍，分攤於多年交付週期。此訂單不僅體量巨大，且Bloom已表明「Oracle只是開始，超過半數backlog來自其他hyperscalers」，隱含後續項目的乘數效應。

③

MTAR產能擴張時程與Bloom需求完美銜接

MTAR正執行分階段產能擴張：Hot Box從8,000台/年→12,000台（FY26底）→20,000台（FY27）→30,000台（FY28）。Bloom法說會確認「持續以每季數百MW的速度擴產」，且KR Sridhar明確表示生產節奏由客戶綠地廠區建設速度決定。這與MTAR的擴產節奏形成天然對接，MTAR若能如期完成產能爬坡，將直接支撐MOFSL預估的FY25–28 Revenue CAGR 40% / EBITDA CAGR 55%。

④

Bloom強調的「不受勞工瓶頸制約」降低對MTAR的執行風險擔憂

法說會中KR明確表示Bloom的自動化製造使工廠擴產10倍但人力不變，且同樣的「Bloom Way」哲學已灌輸至供應鏈夥伴（即MTAR等），強調供應鏈夥伴同樣採用自動化應對超高速擴產。這在一定程度上降低了市場對MTAR執行風險的擔憂，對MTAR股票具有修復估值溢價的潛力。

⑤

Bloom服務合約黏性（10–15年）強化MTAR長期訂單能見度

Bloom 100% service attach rate加上數據中心平均10–15年服務合約，意味著Bloom的設備裝機容量在長達十年以上的週期內均需持續維護，Hot Box Assembly存在折舊後更換的長期需求。這進一步鞏固MTAR未來多年的預期收入基礎，超越單純的「新增裝機」邏輯。

⚠ 主要風險提示

  • 客戶集中度：Bloom佔MTAR營收55–65%，Bloom任何訂單延誤、技術規格轉換或供應商多元化策略均會直接衝擊MTAR財務。
  • 產能爬坡執行風險：從8,000台急速擴至30,000台需要精密的製程管控與設備採購，若交付時程滑移，可能造成Bloom供應缺口，進而影響Bloom收入確認節奏。
  • 地緣政治/關稅風險：印度出口至美國若遭遇關稅衝擊（Bloom法說會有提及「exogenous factors」保留條款），可能壓縮MTAR的美元計價接單吸引力或影響交付計劃。
  • 技術路線演變：Bloom若推動800V DC直供、削減外部組件需求，長期而言可能影響部分周邊組件（如熱管理系統）的採購結構。
  • MTAR自身估值：市場已部分price-in Bloom成長，需評估當前股價是否已充分反映上述正面催化劑。
06 總結

Bloom Energy Q1 2026是公司史上最強的一個季度，Revenue年增130%、Non-GAAP EPS超出Consensus達238%，EBITDA擊敗預期逾164%。更重要的是，公司將全年指引中位數從60%成長大幅上調至80%，並同步披露Oracle 2.45 GW的超大訂單——這兩個訊息共同確立了Bloom作為AI基礎設施電力供應商「標準選擇」的定位，而非僅是一個利基方案。

對MTAR Technologies而言，Bloom本次財報的意義在於：訂單能見度的大幅提升（Bloom backlog已超$20B）、全年Revenue成長加速（中位數+80% YoY），以及Oracle超大項目帶來的多年期巨量Hot Box需求，三者共同強化了MTAR作為Bloom「不可或缺使能者」的戰略價值。若MTAR能如期完成FY26–28的產能爬坡，FY25–28的Revenue/EBITDA/PAT CAGR目標（40%/55%/78%）具備實現的基礎條件。投資者需持續追蹤MTAR各季度的訂單流入（特別是來自Bloom的占比）、Hot Box生產交付量，以及Bloom單季出貨是否持續維持高水位，作為核實投資論點的關鍵指標。
